# 排版美化示例集

本文档通过实例展示如何将原始文本美化为Hexo格式的Markdown文档。

## 示例1：科技教程类文章

### 原始文本

```
BM算法是一种字符串匹配的算法。与KMP相比，BM算法不扫描全部输入字符。
要使用BM算法，需要知道两个信息：用于坏字符规则的bc数组和用于好后缀规则的gs数组。

坏字符规则分为两种情况：
情况1：失配位置指向的文本串中对应的字符，不存在于模式串中。直接将整个模式串移动到失配位置之后。
情况2：失配位置指向的文本串中对应的字符，存在于模式串中，且在失配位置的左边。将模式串中的文本串中对应的字符放在失配位置上。

好后缀规则分为3种情况...
```

### 美化后

```njk
{% note primary %}
## BM算法
{% endnote %}

BM算法是一种字符串匹配的算法。与KMP相比，BM算法不扫描全部输入字符，平均匹配时间c·n，常量 c <1（随机或真实文本），但最坏情况是O(n·m)。

要使用BM算法，需要知道两个信息：
**1.用于坏字符规则的bc数组**
**2.用于好后缀规则的gs数组**

{% note info %}
### 坏字符规则
{% endnote %}

坏字符规则分为两种情况：

1. 失配位置指向的{% label 文本串中对应的字符 red %}，不存在于模式串中。

![坏字符情况1](study/huai2.png)

如上图所示，在这种情况下，直接将整个模式串移动到{% label 失配位置 red %}之后。

2. 失配位置指向的{% label 文本串中对应的字符 red %}，存在于模式串中，且在{% label 失配位置 red %}的左边。

![坏字符情况2](study/huai1.png)

如上图所示，在这种情况下，将模式串中的{% label 文本串中对应的字符 red %}放在{% label 失配位置 red %}上。
```

### 美化要点

- 使用 `{% note primary %}` 包裹总标题
- 使用 `{% note info %}` 包裹子章节
- 使用 `{% label text red %}` 高亮关键术语
- 使用 `**粗体**` 强调要点
- 使用有序列表整理步骤

## 示例2：世界观设定类文章

### 原始文本

```
逝痕文明-前时代 TO:2003
在这一时期，逝痕文明回绝了智能机械的请求。无比漫长的，直达旧日定下的封锁线。
内容包含主线：云末王冕。

云末王冕
导言：这段故事乃是万千宇宙中无数相似开局的一瞬...
概述：云末王冕记载着攸羽在未来世界的一场奇幻的旅途...
定位：作为第九边缘文化体系的开端，本文旨在以泛用的方式...
结构：本文采用分节叙事的结构。
```

### 美化后

```njk
{% note danger %}
### 逝痕文明-前时代 TO:2003
`在这一时期，逝痕文明回绝了智能机械的请求`
`无比漫长的，直达旧日定下的封锁线`
{% endnote %}

内容包含
主线：<云末王冕>

{% note info WORLD_LINE：云末王冕 %}
#### [云末王冕](https://example.com/posts/21.html)
**导言**：这段故事乃是万千宇宙中无数相似开局的一瞬...

**概述**：《云末王冕》记载着攸羽在"未来世界"的一场奇幻的旅途...

**定位**：作为第九边缘文化体系的开端，本文旨在以"泛用"的方式...

**结构**：本文采用<分节叙事的结构>。
{% endnote %}
```

### 美化要点

- 使用 `{% note danger %}` 标记时代分隔
- 使用 `` ` `` 行内代码包裹关键短句
- 使用 `{% note info LABEL %}` 为内容块添加分类标签
- 使用 `<名称>` 格式标记内容类型
- 使用 `**导言**：` 等格式组织信息

## 示例3：Word文档转MD（无标准标题样式）

### 原始Word内容（使用模拟标题）

```
第三章：系统架构设计

3.1 总体架构

本系统采用微服务架构，主要包括以下几个模块：

用户服务：负责用户注册、登录、权限管理等功能。
内容服务：负责文章的创建、编辑、发布和管理。
评论服务：负责评论的发表、审核和展示。

3.1.1 技术选型

后端框架：Spring Boot 3.x
数据库：MySQL 8.0 + Redis 7.0
前端框架：Vue 3 + Element Plus
```

### 美化的识别过程

1. **`第三章：系统架构设计`** → 一级特征（章节标记），作为 `##` 标题，包裹在 `{% note primary %}` 中
2. **`3.1 总体架构`** → 一级特征（编号前缀），作为 `###` 标题，包裹在 `{% note info %}` 中
3. **`用户服务：`** → 三级特征（粗体分类词），不作为标题，仅用 `**粗体**` 强调
4. **`3.1.1 技术选型`** → 一级特征（编号前缀），作为 `####` 标题，包裹在 `{% note warning %}` 中
5. **`后端框架：`** → 非标题，作为列表项

### 美化后

```njk
{% note primary %}
## 第三章：系统架构设计
{% endnote %}

{% note info %}
### 3.1 总体架构
{% endnote %}

本系统采用微服务架构，主要包括以下几个模块：

- **用户服务**：负责用户注册、登录、权限管理等功能。
- **内容服务**：负责文章的创建、编辑、发布和管理。
- **评论服务**：负责评论的发表、审核和展示。

{% note warning %}
#### 3.1.1 技术选型
{% endnote %}

| 层级 | 技术 | 版本 |
|------|------|------|
| 后端框架 | Spring Boot | 3.x |
| 数据库 | MySQL + Redis | 8.0 + 7.0 |
| 前端框架 | Vue 3 + Element Plus | - |
```

### 美化要点

- Word中的"第X章"转为 `{% note primary %}` 包裹的 `##` 标题
- 小节标题使用 `{% note info %}` 包裹
- 服务列表使用无序列表 + `**粗体**` 突出服务名
- 技术栈信息使用表格呈现

## 示例4：弹窗块使用场景

```html
<button id="btn-weifuwu" class="btn-beautify button--animated" title="点击了解微服务架构">🏗️ 点我了解：微服务架构</button>

      icon: "info",
      button: "知道了",
    });
  });
})();
</script>
```

## 排版通用原则

1. **层级清晰**：`##` → `###` → `####` → `#####`，最多5级正文标题，禁止跳级
2. **视觉分隔**：不同主题间使用 `------` 或 `{% note danger %}` 分隔
3. **重点突出**：关键术语使用 `**粗体**` 或 `{% label %}` 或弹窗块
4. **数据规整**：结构化数据优先使用表格；3行以上重复的"名称：描述"模式转为表格
5. **步骤明确**：操作步骤使用有序列表
6. **适度留白**：段落间保留空行，标题/代码块/表格/列表前后各留一个空行
7. **信息归类**：相关内容用note块包裹，形成视觉组
8. **弹窗辅助**：每个重要术语首次出现时考虑添加弹窗解释
9. **空格规范**：中英文间加空格，中文与数字间加空格，全角标点与其他字符不加空格
10. **每行80-100字符**：便于版本控制和多屏阅读
11. **文末附表**：根据文章类型附加总结、术语表、操作速查等概括性内容
12. **严格忠实原文**：不删除、不修改原文任何文字，仅优化排版格式

## 示例6：文末附表 — 技术教程类

对于技术教程或知识科普类文章，在文末附加**内容总结** + **操作速查**：

```njk
{% note success %}
## 📋 内容总结

- **核心主题**：BM算法是一种高效的字符串匹配算法，通过"坏字符规则"和"好后缀规则"实现跳跃式匹配
- **关键概念**：
  - **坏字符规则**：根据失配字符在模式串中的位置决定跳跃距离
  - **好后缀规则**：利用已匹配的后缀信息进行安全跳跃
- **主要结论**：
  1. BM算法平均时间复杂度为 O(n/m)，优于KMP
  2. 实际应用中BM算法是字符串匹配的首选
{% endnote %}

{% note warning %}
## 🛠️ 操作速查

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 构建 bc 数组 | 记录每个字符在模式串中最右出现位置 |
| 2 | 构建 gs 数组 | 记录每个后缀的最右匹配位置 |
| 3 | 从右向左匹配 | 文本串与模式串右端对齐，逐字符比较 |
| 4 | 失配时跳跃 | 取 bc 和 gs 中更大的跳跃距离 |

> **核心公式**：`shift = max(bc[text[i]] - (m-1-i), gs[i])`
{% endnote %}
```

## 示例8：文末附表 — 常见问题（FAQ）

对于使用说明或教程类文章，使用折叠块收纳常见问题：

```html
<details>
<summary>📋 常见问题（点击展开）</summary>

**Q：为什么执行 `hexo g` 后文章不显示？**
A：检查 frontmatter 中的 `date` 格式，必须是 `YYYY-MM-DD HH:mm:ss`。

**Q：图片无法加载？**
A：确认图片放在 `source/images/` 目录下，引用路径使用 `images/xxx.png`。

**Q：note块不显示样式？**
A：确认主题（Butterfly/NexT）支持 note 标签插件。

</details>
```

# Word转MD深度指南

本文档详细说明将Word文档转换为Hexo Markdown时的完整工作流程、常见陷阱与解决方案。

## 一、转换前分析：Word文档状况评估

在开始转换前，先评估Word文档的"健康度"：

### 健康文档（理想情况）
- 标题使用了内置样式（"标题1"、"标题2"...）
- 正文使用"正文"样式
- 列表使用了自动编号功能
- 图片有正确的位置和说明

### 病态文档（常见情况）
- 标题仅靠字体放大+加粗模拟
- 无一致的样式应用
- 列表为手动输入的数字
- 图片嵌入位置随意

**无论文档健康状况如何，都必须逐行审视，不遗漏任何内容。**

## 二、逐行审查流程

### 第1轮：全局结构扫描

先通读全文，建立整体结构认知：

1. 标记所有可能的标题行（按 `SKILL.md` 中的三级特征规则）
2. 确定标题层级关系（哪个是 `##`，哪个是 `###`，哪个是 `####`）
3. 建立树状结构草图

### 第2轮：逐段转换

按结构草图，逐段处理：

1. 先写标题（加上对应note块包裹）
2. 再写正文（按排版规则美化）
3. 识别并添加弹窗块
4. 处理列表、表格

### 第3轮：全文审查

1. 检查标题层级是否连贯（不跳级）
2. 检查中英文空格是否正确
3. 检查是否有遗漏的原文内容
4. 检查note块是否正确闭合

## 三、隐式标题识别实战

### 场景1：中文编号系统

```
一、项目背景          → ## 一、项目背景
（一）市场分析        → ### （一）市场分析
1. 目标用户           → #### 1. 目标用户
（1）年龄分布         → **（1）年龄分布**：（非标题，用粗体）
```

注意：
- `一、` `二、` 开头 → `##`
- `（一）` `（二）` 开头 → `###`
- `1.` `2.` 开头 → `####`  
- `（1）` `（2）` 开头 → **粗体列表**，不作为标题

### 场景2：数字编号系统

```
1 概述                → ## 1 概述
1.1 背景              → ### 1.1 背景
1.1.1 问题描述         → #### 1.1.1 问题描述
1.1.1.1 具体情况       → **1.1.1.1 具体情况**：（非标题，用粗体）
```

### 场景3：无编号的隐式标题

```
（居中、加粗、大号字体）
系统架构设计           → ## 系统架构设计

（独行短句，前后空行）
微服务架构的优势        → ### 微服务架构的优势

（粗体开头的段落）
性能优化策略           → **性能优化策略**：（非标题，粗体分类词）
```

### 场景4：混合模式

当一篇文档同时使用多种编号风格时，需要判断其层次关系：

```
第一章 绪论           → ## 第一章 绪论
1.1 研究背景          → ### 1.1 研究背景
一、国内外现状         → ### 一、国内外现状（与1.1同级）
（一）国外研究         → #### （一）国外研究
（二）国内研究         → #### （二）国内研究
二、本文结构           → ### 二、本文结构
```

## 四、常见转换陷阱

### 陷阱1：混淆标题和列表项

```
错误：
- 标题A           → 标题A（误标为列表项）
- 标题B           → 标题B（误标为列表项）

正确：
## 标题A
- 列表项1
- 列表项2

## 标题B
- 列表项1
- 列表项2
```

**判断方法**：如果"列表项"是概括性的名称（如模块名、功能名），且其下有详细描述，则更可能是标题。

### 陷阱2：编号层次错位

```
错误：
一、概述              → ## 一、概述 ✅
1.1 背景              → ## 1.1 背景 ❌（应为 ###）

正确：
一、概述              → ## 一、概述
1.1 背景              → ### 1.1 背景
```

### 陷阱3：图片丢失

Word中的图片无法直接嵌入Markdown。处理方式：
1. 将Word文档另存为 `.zip`，解压后从 `word/media/` 提取图片
2. 将图片放置到Hexo的 `source/images/` 目录
3. 在MD中以 `![描述](images/文件名)` 引用

### 陷阱4：复杂表格损坏

带有合并单元格的表格在Markdown中难以呈现。处理方式：
1. 简单表格 → 直接使用Markdown表格语法
2. 复杂表格 → 使用HTML `<table>` 标签
3. 极复杂表格 → 截图保存为图片，以图片形式嵌入

### 陷阱5：Word自动编号转MD

Word的自动编号在转换时常丢失。需要：
1. 从Word文档中读出实际的编号文本
2. 在MD中手动重建编号结构
3. 或使用Markdown的 `1.` 自动编号（让渲染器自动递增）

## 五、Hexo验证

转换完成后执行：

```bash
hexo clean && hexo generate
```

常见报错及解决：

| 错误现象 | 可能原因 | 解决方式 |
|---------|---------|---------|
| `YAMLException` | frontmatter格式错误 | 检查 `: ` 后是否有空格、缩进是否正确 |
| `Template render error` | note/label等标签语法错误 | 检查标签是否闭合、参数是否正确 |
| 文章不显示 | `date` 格式错误 | 确保格式为 `YYYY-MM-DD HH:mm:ss` |
| 图片不显示 | 图片路径错误 | 确保图片在 `source/images/` 下，引用路径为 `images/xxx.png` |

## 六、HTML `<img>` 标签精细控制

当需要精确控制图片大小、圆角、阴影时，使用HTML代替Markdown图片语法：

```html
<!-- 小巧精致风 -->
<img src="images/demo.png" alt="演示图" width="20%" style="border-radius: 8px; border: 2px solid #ccc;" />

<!-- 质感悬浮风 -->
<img src="images/demo.png" alt="演示图" width="50%" style="border-radius: 12px; box-shadow: 0 4px 20px rgba(0,0,0,0.15);" />
```

## 七、折叠块（长内容收纳）

对于过长的代码示例、安装步骤等，使用折叠块：

```html
<details>
<summary>点击展开查看详细安装步骤</summary>

1. 下载安装包...
2. 配置环境变量...
3. 验证安装...

</details>
```

## 八、转换检查清单

转换完成后，逐项确认：

- [ ] frontmatter格式正确，`date`/`updated` 时间格式正确
- [ ] 所有标题层级连贯，无跳级
- [ ] `{% note %}` 标签正确闭合
- [ ] 中英文之间有空格
- [ ] 中文与数字之间有空格
- [ ] 专有名词大小写正确
- [ ] 所有原文内容均已保留，无遗漏
- [ ] 图片路径正确
- [ ] 弹窗块按钮有唯一的id
- [ ] `hexo clean && hexo generate` 无报错
